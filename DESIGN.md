# SCADA Design Language

本文定义当前 SCADA Web 端的统一视觉规范。目标不是做通用设计教程，而是为后续页面开发、重构和安装器界面延展提供一套可直接执行的工程标准。

## 1. 总体气质

- 产品类型：工业监控 / 工具型系统，不是营销站，不是内容站。
- 视觉关键词：稳定、紧凑、清晰、低装饰、强信息层级。
- 页面优先服务扫描效率，不用无意义的大段说明文案。
- 品牌感主要来自侧边栏、标题栏、颜色和秩序，不来自装饰插画。
- 默认删除无价值副标题；如果副标题不能提升理解，就不显示。

## 2. 字体规范

全局字体必须优先使用统一字体变量：

```css
--app-font-family: "HarmonyOS Sans SC Medium", "HarmonyOS Sans SC", "Microsoft YaHei", "PingFang SC", sans-serif;
```

适用范围：

- `html`
- `body`
- `#root`
- `button`
- `input`
- `select`
- `textarea`
- `table`
- 所有业务页面容器，包括报表页、配置页、数据看板、运行时页面和安装器参考界面。

字体规则：

- 页面主标题：`700`
- 卡片主标题：`700`
- 导航文字：常规 `500-600`，激活态可提升到 `600+`
- 正文：`400`
- 标签、字段名、表头：`600-800`
- 不使用不存在的 `HarmonyOS Sans SC Bold` 作为硬依赖。
- 不在单个模块里硬编码 `Microsoft YaHei UI` 覆盖全局字体，除非是 WinForms 安装器这类非 Web 平台。

## 3. 核心颜色

- 页面底色：`#f5f7fb` 或邻近浅灰蓝。
- 内容底色：`#ffffff`
- 结构描边：`#e6ebf4`
- 正文主文字：`#1f2937`
- 次级文字：`#6b7280`
- 主操作蓝：`#4263eb`
- 侧边栏主蓝：`#233264`
- 侧边栏深蓝终点：`#1d2a56`
- 侧边栏激活背景：`#2f4acb`
- 数据看板强调蓝：`#005f87`

状态色：

- 正常 / 运行：`#2eaa4a`
- 待机 / 警示：`#ecb823`
- 故障 / NG：`#ca3333`
- `NG` 必须使用红色语义，并搭配圆点、底色或边框，不只靠文字颜色表达。

## 4. 侧边栏

- 展开宽度：`216px`
- 收起宽度：`72px`
- 背景：深蓝到更深蓝的纵向渐变。
- 导航项：矩形、无大圆角。
- 激活态：实色蓝底，左侧 `3px` 白色竖条。
- hover：低透明白色背景，文字切换为白色。
- 登录 / 注销入口视为一等导航成员，样式与导航体系一致。

## 5. 标题栏

- 标题栏整体紧凑，不做营销 Hero。
- 默认白底或浅底。
- 与主体内容之间靠轻微层次分隔，不靠厚重描边。
- 默认结构为英文 kicker + 中文主标题；纯报表页面可只保留业务报表标题。
- 副标题默认删除，只有确实有操作价值时保留。

主标题样式：

- 字号约 `1.45rem`
- 字重 `700`
- 常规颜色：`#1f2a3b`
- 对齐数据看板的重点页面可使用 `#005f87`

## 6. 卡片与边界

- 默认不使用明显框线或重描边。
- 卡片边界优先通过轻阴影、留白、背景色差表达。
- 仅在表格、输入框、状态区等需要精确边界的组件上使用浅描边。
- 同一页面内卡片边界策略必须统一，避免有的靠边框、有的靠阴影。
- 工业工具界面默认不保留圆角；卡片、工具栏、按钮、输入框优先使用直角。只有明确有业务或平台一致性理由时，才允许使用很小圆角。

## 7. 表单与按钮

输入框：

- 业务表单输入框、下拉框、文本域统一沿用返修管理输入规范。
- 高度：常规表单控件 `38px`；紧凑表格内编辑控件可降到 `32px - 34px`，但视觉语言保持一致。
- 背景：`#ffffff`
- 边框：`1px solid #cbd5e1`
- 圆角：`0`，输入框、下拉框、文本域不允许圆角。
- 内边距：单行输入 `0 12px`；文本域 `8px 10px`
- 文字：`#1e293b` 或 `#1f2937`
- 字段标签：`0.8rem`、`600`、`#475569`
- Focus：取消默认 outline，边框使用 `#2563eb`，并添加 `0 0 0 3px rgba(37, 99, 235, 0.12)` 外发光。
- 同一页面内所有新增/编辑表单必须使用这一套输入规范，不允许混用内阴影输入框、深色输入框或无边框输入框。

按钮：

- 主按钮：`#4263eb` 背景，白字，字重 `600`
- 次按钮：白底、灰蓝边框、深灰蓝文字
- 风险按钮：红色语义
- 工具栏优先使用图标 + 短文字，不做长串文字按钮。

## 8. 表格

- 表格是核心工作面，不应花哨。
- 重点是列宽分配、对齐、可扫描性。
- 高密度业务表格允许紧凑，但必须整齐、可读。
- 内容区如用户明确要求“无框线”，应取消单元格内部框线，用斑马纹或留白区分行。
- 表头可以使用蓝底白字，但不要使用伪元素制造悬浮芯片，除非该样式已经被明确确认。

## 9. 数据看板语言

数据看板是整套系统的视觉源头之一，后续页面可以借用视觉语言，但不要机械复制业务布局。

可继承：

- 蓝 / 黄 / 红 / 绿的状态语义
- 指标标题前的细色条
- 工位标题的蓝色表达
- 高密度信息组织
- 高辨识度报警提示

不可滥用：

- 不要把所有页面都做成工位板卡布局。
- 不要把看板里的状态组件硬套到普通配置页。

## 10. 动效

- 动效只服务于状态变化和空间切换。
- 默认时长短促，不拖沓。
- 侧边栏主过渡：`220ms - 240ms`
- 快速淡出：`140ms`
- 报警进入短且明确，退出更平顺但不能卡顿。

## 11. 一句话总结

这套 SCADA UI 的核心是：用稳定的深蓝侧边栏、紧凑的标题栏、统一的 HarmonyOS 字体体系、浅灰蓝工作底色和高信息密度卡片表格，构建一套面向工业场景的专业操作界面。
