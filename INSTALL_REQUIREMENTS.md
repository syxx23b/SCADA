# SCADA 安装要求整理

## 一、发布与部署要求

- 项目发布目标为 Windows 环境。
- 前端构建产物需要输出到 `Scada.Api/wwwroot`。
- API 默认以 `5000` 端口对外提供服务。
- 部署配置要求从当前项目根目录构建，并部署到 `http://localhost:5000`。
- 安装包默认使用 API 的发布目录作为 payload 来源：
  `Scada.Api/bin/Release/net8.0/win-x64/publish`

## 二、服务安装要求

- 安装程序只能在 Windows 上运行。
- 安装程序必须以管理员权限运行。
- 安装时需要先停止并删除旧服务。
- 安装目录默认位于：
  `$(ProgramFiles)\Scada_Codex_Siemens`
- 默认服务名为 `ScadaApi`。
- 默认服务显示名为 `SCADA API`。
- 服务启动方式为 Windows Service。
- 服务运行账户为 `LocalSystem`。
- 服务启动参数应包含：
  `--urls http://0.0.0.0:5000`
- 服务注册后应设置为开机自启。
- 服务失败后应自动重启，重启间隔为 5000ms。
- 安装结束后应自动启动服务。
- 卸载时应删除服务、删除防火墙规则、删除安装目录。

## 三、防火墙要求

- 安装完成后应自动配置 Windows 防火墙入站规则。
- 默认放行 TCP `5000` 端口。
- 规则名称应与服务显示名和端口绑定。

## 四、安装界面要求

- 安装器使用 Windows Forms。
- 窗口为固定大小，不可最大化，不可最小化。
- 窗口初始居中显示。
- 界面字体为 `Microsoft YaHei UI`。
- 顶部应有品牌横幅、标题和提示信息。
- 界面应提供以下可配置项：
  - 安装目录
  - 服务名称
  - 显示名称
  - 监听端口
- 安装目录应支持浏览选择。
- 端口输入应使用数字控件，范围为 `1-65535`。
- 界面底部应提供以下按钮：
  - 安装并启动
  - 卸载
  - 退出
- 应提供日志窗口显示安装过程和命令输出。
- 执行安装或卸载时，按钮应进入忙碌状态并临时禁用。
- 操作成功后应弹出成功提示。
- 操作失败后应弹出错误提示，并把错误写入日志。

## 五、静默安装要求

- 安装器支持 `--silent` 参数。
- 带 `--silent` 启动时，不显示界面，直接使用默认配置执行安装。

## 六、默认信息

- 开发者：`ZhangXC`
- 公司：`松门电器`
- 安装程序产品名：`SCADA 安装程序`
- 安装器默认服务端口：`5000`

## 七、备注

- 当前安装器源码中的中文界面文本存在乱码现象，后续应修复编码后再正式发布。

## 八、安装包命名与输出位置

- 安装包文件名应包含时间戳。
- 命名格式建议为：
  `SCADA-Setup-YYYY.MM.DD.HHMM.exe`
- 示例：
  `SCADA-Setup-2026.04.24.0406.exe`
- 最终安装包应输出到：
  `C:\Users\syxxz\OneDrive\SCADA`
